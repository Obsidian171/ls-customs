# 📋 Инструкция по настройке

## Шаг 1: Supabase

### 1.1 Создание проекта
1. Зайди на [supabase.com](https://supabase.com)
2. Создай новый проект
3. Запомни пароль базы данных

### 1.2 Выполнение SQL
1. Открой SQL Editor в Supabase
2. Скопируй весь код из `supabase_schema.sql`
3. Вставь и нажми RUN
4. Убедись что все таблицы созданы (9 таблиц)

### 1.3 Создание админа
1. Перейди в Authentication → Users
2. Нажми "Add user" → "Create new user"
3. Введи email и пароль (запомни их!)
4. Подтверди email (Auto Confirm User = ON)

### 1.4 Получение ключей
1. Перейди в Settings → API
2. Скопируй:
   - `Project URL` → это твой `VITE_SUPABASE_URL`
   - `anon public` → это твой `VITE_SUPABASE_ANON_KEY`

## Шаг 2: Настройка проекта

### 2.1 Установка зависимостей
```bash
npm install
```

### 2.2 Создание .env
Создай файл `.env` в корне проекта:

```env
VITE_SUPABASE_URL=https://xxxxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
VITE_ADMIN_SECRET=my-secret-admin-path-2024
```

⚠️ **Важно**: 
- `VITE_ADMIN_SECRET` — это путь к админке (например `x-panel` или `admin-2024`)
- Не используй простые слова типа `admin`, `panel`
- Админка будет доступна по адресу: `http://localhost:5173/{VITE_ADMIN_SECRET}/login`

## Шаг 3: Запуск

### Development
```bash
npm run dev
```

Сайт откроется на `http://localhost:5173`

### Вход в админку
1. Открой `http://localhost:5173/{твой-VITE_ADMIN_SECRET}/login`
2. Введи email и пароль из Supabase Auth
3. Готово!

### Регистрация пользователя
1. Открой `http://localhost:5173/register`
2. Заполни форму регистрации
3. Подтверди email (если включено в Supabase)
4. Войди через `/login`
5. Доступ к личному кабинету: `http://localhost:5173/cabinet`

## Шаг 4: Настройка Email Auth в Supabase

### 4.1 Включение Email Auth
1. Перейди в Authentication → Providers
2. Убедись что Email включен
3. Настрой Email Templates (опционально)

### 4.2 Настройка Redirect URLs
1. Перейди в Authentication → URL Configuration
2. Добавь Site URL: `http://localhost:5173` (для dev)
3. Добавь Redirect URLs:
   - `http://localhost:5173/**`
   - `https://твой-домен.com/**` (для production)

### 4.3 Отключение подтверждения email (для тестирования)
1. Перейди в Authentication → Settings
2. Найди "Enable email confirmations"
3. Отключи для быстрого тестирования
4. ⚠️ Включи обратно для production!

## Шаг 5: Заполнение контента

### Через админку
1. Войди в админку
2. Добавь услуги (Services)
3. Добавь сотрудников (Employees)
4. Добавь фото в галерею (Gallery)
5. Добавь вакансии (Vacancies)
6. Обнови контакты (Contacts)

### Fallback данные
Если таблицы пустые, на некоторых страницах показываются демо-данные (Services, Team, Gallery).

## Проблемы?

### Ошибка "Missing Supabase environment variables"
- Проверь что `.env` файл создан
- Проверь что переменные начинаются с `VITE_`
- Перезапусти dev-сервер

### Ошибка при входе в админку
- Проверь что пользователь создан в Supabase Auth
- Проверь что email подтверждён
- Проверь правильность email/пароля

### Таблицы не создались
- Выполни SQL заново
- Проверь что нет ошибок в SQL Editor
- Проверь что RLS включен (должен быть)

### Rate limit срабатывает сразу
- Очисти localStorage в браузере
- Или измени лимиты в `src/lib/rateLimit.ts`

### Не могу войти в личный кабинет
- Проверь что пользователь зарегистрирован
- Проверь что email подтверждён (или отключи подтверждение)
- Проверь что таблица `user_profiles` создана
- Проверь RLS политики для user_profiles

### Не вижу свои заявки в кабинете
- Проверь что заявки создавались после входа в систему
- Проверь что user_id правильно сохраняется
- Проверь RLS политики для bookings и part_orders

## Личный кабинет пользователя

### Функционал
- **Профиль** - редактирование личных данных
- **Мои записи** - история записей на обслуживание
- **Заказы деталей** - отслеживание заказов запчастей
- **Отзывы** - создание и редактирование отзывов
- **Уведомления** - получение уведомлений от администрации

### Автозаполнение форм
Когда пользователь авторизован:
- Форма записи автоматически заполняет имя и телефон
- Форма заказа деталей заполняет все данные из профиля
- Все заявки привязываются к user_id для отслеживания

### Для администраторов
- В админ-панели видны email и user_id в заявках
- Можно добавлять примечания (admin_notes)
- Модерация отзывов перед публикацией
- Создание уведомлений для пользователей (нужно добавить в админку)

Подробнее см. `USER_CABINET_GUIDE.md`
